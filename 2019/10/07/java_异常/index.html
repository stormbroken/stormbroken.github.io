使用异常机制的建议

  1. 异常声明是API的一部分
  2. 异常处理不能代替简单的测试
  3. 不要过分地细化异常
  4. 利用异常层次结构
    • 不要只抛出RuntimeException，应该寻找更合适的子类或者创建自己的异常类
  5. 不要压制异常
  6. 在检测错误时，苛刻比放任要好
    • 比如在错误的时候抛出EmptyStackException比抛出NullPointerException异常更好
  7. 不要羞于传递异常
    • 早抛出，晚捕获
